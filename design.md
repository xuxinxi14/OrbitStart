你现在要帮我把 OrbitStart 的 GUI 做成一套完整的「Local Galaxy / 本地星系」风格界面。

项目背景：
OrbitStart 是一个原创的、可扩展的 Windows 启动工作台，定位是 local-first launcher。它可以管理应用、网址、文件、工作区、脚本、插件、主题、动作链等资源。当前功能基本完整，但界面还是偏普通 Web UI / 管理后台风格。我希望它升级成一个有独特品牌感、高级感和整体感的桌面 GUI。

核心目标：
在不破坏现有功能逻辑的前提下，把当前主界面重构为「深邃、复古、唯美、丝滑」的 Local Galaxy 风格。

请先完整阅读项目结构，理解当前前端技术栈、组件结构、状态管理和样式写法。不要盲目重写整个项目。优先在现有架构上做视觉升级和组件整理。

我会同时提供两张图片：一张是当前 OrbitStart 主界面截图，另一张是目标 Local Galaxy GUI 参考图。请以当前截图为布局基础，以参考图为视觉目标，不要照搬参考图中的错误文字或不可实现细节。

========================

设计方向名称：
Local Galaxy / 本地星系

关键词：

- 深邃
- 复古
- 唯美

整体感觉：
OrbitStart 不再像普通启动器，而像一个「个人本地资源星系导航站」。用户的应用、网址、文件、工作区、脚本、插件都像星系中的资源节点，可以被搜索、收藏、启动和组织。

视觉上要像一个精致的星际导航控制台，但不能变成廉价赛博朋克，也不能只是普通毛玻璃。要有复古天文仪、星盘、星图、金属徽章、深空、暗金线条、柔和星光的感觉。

参考气质：

- 深色高级桌面软件
- 复古星图 / 星盘 / 天文仪
- 古典金色细线 + 深蓝黑宇宙背景
- 丝滑的曲线和光晕
- 信息密度高但不杂乱
- 真实可用，不是纯概念海报

========================
五、背景设计
========================

主界面背景需要有深空感，但不能干扰阅读。

请实现：

1. 深蓝黑渐变背景。
2. 顶部或中部有非常淡的星云光晕。
3. 背景上有极淡的星点。
4. 少量金色星图线条 / 轨道线 / 圆弧装饰。
5. 这些装饰必须 pointer-events: none，不影响交互。
6. 装饰层透明度要低，不能让界面变乱。

可以创建一个全局背景层组件，例如：
GalaxyBackdrop / CosmicBackdrop。

它包含：

- radial-gradient 星云
- pseudo-element 星点
- SVG 或 CSS 绘制的轨道弧线
- 少量柔和光斑

注意：
背景不能导致性能明显下降，不要使用复杂 canvas 动画。优先使用 CSS gradient、SVG、伪元素。

十、资源卡片设计
========================

资源卡片是核心组件，请重点优化。

卡片目标：
像「本地星系中的资源节点卡片」，既高级又可读。

结构保持：

- 左上资源图标

- 路径 / URL / 简介
- 启动次数 / 最近启动时间
- 底部操作：收藏、编辑、删除

视觉要求：

1. 卡片背景为深蓝黑半透明。
2. 边框为暗金色或低透明浅色。
3. hover 时出现细微青绿色光晕。
4. 图标容器统一成圆角方形或圆形星体舱。

   - 名称：米白 / 暖白，稍粗

7. 卡片内可以有极淡的弧线纹理，但不能影响内容阅读。

尺寸建议：

- 保持当前网格密度，不要让卡片太大。
- 但适度增加内边距，让视觉不拥挤。
- 卡片高度略微统一，避免信息跳动。

hover 动效：

- transform: translateY(-2px)
- border-color 增强
- box-shadow 增强
- 图标容器轻微发光

十三、字体与排版
========================

中文字体优先使用系统字体，不要引入难以部署的外部字体。

推荐：
font-family:
"Inter", "Segoe UI", "Microsoft YaHei UI", "Microsoft YaHei", system-ui, sans-serif;

标题可以稍微更有气质，但不要使用难读字体。
如果项目无法引入 Inter，就用系统字体。

- 页面大标题：28-34px，font-weight 700
- 小标签：11-12px，letter-spacing 0.06em
- 卡片标题：14-16px，font-weight 600
- 路径 / 元信息：11-12px
- 按钮：13-14px

========================
十四、动效要求
========================

整体要「丝滑」，但动效必须克制。

实现：

- hover 120-180ms ease-out
- focus 160ms ease-out
- 卡片 hover 轻微浮起
- 搜索框 focus 柔和发光
- 按钮 hover 轻微提亮
- 页面进入时可以轻微 fade-in，但不要复杂动画

不要：

- 大量粒子动画
- 快速闪烁
- 强烈霓虹呼吸
- 卡片乱飞
- 影响性能的动画
