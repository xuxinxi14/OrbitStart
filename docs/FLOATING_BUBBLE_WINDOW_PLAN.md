# Floating Bubble Window

本文档记录 OrbitStart 悬浮启动球的当前实现。

## 用户行为

- 启用悬浮启动球后，主窗口隐藏时会显示桌面悬浮球。
- 左键点击悬浮球会恢复 OrbitStart 主窗口。
- 右键点击悬浮球会打开快捷菜单。
- 开启“鼠标悬浮展开快捷菜单”后，鼠标停留在悬浮球上会打开快捷菜单。
- 拖动悬浮球可以移动位置；开启“靠边吸附”后，松手会贴近屏幕左右边缘。

## 窗口结构

悬浮球由两个独立窗口组成：

- `floating-bubble`：收起态主球窗口，使用圆形原生 Region。
- `floating-bubble-menu`：快捷菜单窗口，使用圆角胶囊原生 Region。

主球窗口不再承载展开按钮，因此不会被拉伸成大尺寸透明矩形。快捷菜单由独立窗口承载，显示搜索、添加资源、工作区、最近项目和设置入口。

## 视觉贴图

悬浮球视觉由 `public/design/` 下的 PNG 贴图驱动：

- `大悬浮球(无光晕).png`：主悬浮球默认状态。
- `大悬浮球(有光晕).png`：主悬浮球鼠标悬停状态。
- `小悬浮球(无光晕).png`：快捷菜单按钮默认状态。
- `小悬浮球(有光晕).png`：快捷菜单按钮鼠标悬停状态。

CSS 不再额外绘制主球背景、边框和缩放阴影，避免覆盖贴图或造成边缘裁切。

## Windows 原生 Region

Windows 桌面端通过 Rust/Tauri 调用 Win32 Region API：

- `CreateEllipticRgn` 用于主球窗口。
- `CreateRoundRectRgn` 用于快捷菜单窗口。
- `SetWindowRgn` 将 Region 应用到原生窗口。

Region 基于窗口物理像素尺寸计算。窗口大小变化后会重新应用 Region。

## 悬停与拖动

- 鼠标停留在主球上超过设置的延迟后打开快捷菜单。
- 鼠标从主球移动到快捷菜单时保留短暂缓冲，避免经过窗口间隙时菜单立即收起。
- 按住左键拖动主球时，悬停监控不会弹出或隐藏快捷菜单。
- 快捷菜单窗口内的按钮可直接点击，点击后恢复主窗口并执行对应入口动作。

## 设置项

设置入口：

```text
设置中心 -> 悬浮球
```

可配置项：

- 启用悬浮启动球
- 主窗口隐藏时显示悬浮球
- 悬浮球始终置顶
- 悬浮球大小
- 靠边吸附
- 鼠标悬浮展开快捷菜单
- 不透明度
- 重置悬浮球位置

## 验收点

- 收起态显示为小圆形窗口。
- 主球窗口外区域不响应点击。
- 快捷菜单按钮可点击。
- 拖动、吸附、置顶和透明度设置生效。
- 主窗口隐藏后是否显示悬浮球由设置控制。
