# OrbitStart 视频脚本事实稿

## 产品定位

OrbitStart 是一个本地优先、可扩展的 Windows 启动工作台。它不是替代系统桌面，而是把应用、网址、文件、文件夹、工作区、脚本、插件和动作入口汇聚到一个统一资源中心。

核心表达：

用户不需要在桌面、开始菜单、浏览器书签和不同文件夹里反复查找入口。OrbitStart 把这些入口按真实任务组织起来，让常用工具和内容可以被搜索、分组、收藏、导入、启动和备份。

## 当前工程状态

从 `E:\OrbitStart\package.json` 看，当前版本为 `0.4.3`。

技术栈：

- Tauri 2
- React 18
- TypeScript

- SQLite

主要面向 Windows 桌面环境。

## 可介绍功能

- 资源中心：管理应用、网站、文件、文件夹、工作区、脚本和动作链。
- 快速搜索：通过名称、路径、别名、标签和插件结果查找资源。
- 标签分组：内置应用、工作区、网址、脚本、插件分组，并支持自定义分组。
- 收藏与统计：记录收藏、启动次数和最近启动时间。
- 批量管理：批量选择资源后移动分组或删除。
- 拖拽创建：把桌面快捷方式、文件、文件夹或脚本拖入窗口创建资源。
- Windows 扫描：扫描桌面和开始菜单快捷方式，并尽量提取真实应用图标。
- 浏览器书签导入：支持从 Edge/Chrome 书签文件导入网址资源。
- 动作链：一个入口按顺序启动多个应用、文件夹或网页。
- 插件管理：查看插件清单、权限、状态、详情，并可启用或停用插件。
- 主题工作室：内置多套主题，默认主题为 Local Galaxy，并支持主题 token 扩展。
- 桌面外壳：自定义标题栏、托盘菜单、全局快捷键、右键菜单和外部链接拦截。
- 数据备份：导出或导入本地 JSON 备份。

## Local Galaxy 视频表达

Local Galaxy 不是单纯深色主题，而是把“本地资源”表达为“本地星系”：

- 应用、文件、网址是资源节点。
- 搜索和命令面板是跃迁入口。
- 插件是引擎。
- 主题工作室是轨道控制的一部分。
- 本地数据和 JSON 备份提供可控感。

视频应避免夸张地宣传“全自动”“云同步”“已经完整隔离第三方插件”“自动更新已完成”等尚未完全成立的能力。

## 已知边界

- 第三方插件执行隔离仍处于早期阶段，当前更接近 manifest 驱动和宿主 API 原型。
- Everything 搜索、窗口切换等高级能力已有插件入口，但还没有完整原生 provider。
- 自动更新、签名和公开 release channel 还未完成。
- 当前主要面向 Windows，其他平台需要额外适配和测试。
